# External Project Workflow

Usagi projects are expected to live one directory above the engine checkout, with
the engine in a `Usagi` subdirectory:

```text
ProjectRoot/
  Usagi/
  ProjectName/
  Data/
  Tools/
  rakefile
  ProjectName.sln
```

## Toolchain

Use the versions named in the root `README.md` unless a local setup script has
already staged compatible tools:

- Visual Studio 2022 with the C++ desktop workload.
- Ruby 2.3.3.
- Python 2.7.14.
- Vulkan loader and SDK headers/libraries.
- FBX SDK only when building or running model conversion through Ayataka.
- Oculus SDK only when building VR support.

Run `Setup.bat` from the engine checkout after installing tools. It installs
legacy Ruby/Python packages and runs `EnvironmentSetup.exe`. Re-run environment
setup whenever switching to a different project so `USAGI_DIR` points at the
engine checkout used by that project.

Setup entry points:

- `Setup.bat`
- `Vendor\InstallPyPackages.bat`
- `Vendor\install_gems.bat`
- `EnvironmentSetup.exe`

Important required environment variables include `USAGI_DIR`, `MSBUILD_DIR`,
`PHYSX_DIR`, `VULKAN_SDK`, `VK_SDK_PATH`, `DXSDK_DIR`, and `PATH`.

Optional or command-specific variables include `FBXSDK_DIR`, `OCULUS_SDK_DIR`,
`USAGI_STL`, `USAGI_STL_INCLUDE`, `USAGI_EABASE_INCLUDE`, `USAGI_STL_LIB`,
`build`, `platform`, `region`, `verbosity`, `project`, `cores`, `embed_dlp`,
`VISUAL_STUDIO_BUILD`, and `VISUAL_STUDIO_PLATFORM`.

## Creating A Project From The Template

From `Usagi\Templates`, run:

```powershell
rake project=ProjectName
```

The template writes files to the parent of the engine checkout. It creates the
project source directory, `Data`, `Tools`, a project `rakefile`, and a Visual
Studio solution. The generated project `rakefile` expects the engine directory to
be named `Usagi`.

After generation, run environment setup and regenerate project files:

```powershell
cd ..\ProjectRoot
.\Usagi\Setup.bat
rake platform=win build=debug,release projects
```

## Common Build Commands

From the external project root:

```powershell
rake platform=win build=debug projects
rake platform=win build=debug data
rake platform=win build=debug build
```

Useful variants:

```powershell
rake platform=win build=release build
rake platform=win build=debug,release data
rake platform=win build=debug clean_data
rake platform=win build=debug clean_projects
```

The generated project `rakefile` delegates engine-side work to
`Usagi\rakefile`, then builds project protobufs, data, and Visual Studio
projects through the generated Ninja files.

Useful engine-only checks from the engine checkout:

```powershell
powershell -ExecutionPolicy Bypass -File .\Tools\Tests\BaselineBuild\Run.ps1
powershell -ExecutionPolicy Bypass -File .\Tools\Tests\ParticleEditorSmoke\Run.ps1 -Launch
```

Use `-RequireValidation` on the Particle Editor smoke runner when a Vulkan SDK
with `VK_LAYER_KHRONOS_validation` is staged under the local tools directory.

## Data Build Inputs

Project data builds use the same pipeline as engine data:

- `Data\Audio` for audio project data.
- `Data\Entities` for entity YAML.
- `Data\GLSL\effects` and `Data\GLSL\shaders` for project shader packages.
- `Data\Models`, `Data\Textures`, `Data\Fonts`, and particle folders for
  converted runtime assets.
- `Data\CustomFX` for custom post-effect definitions.
- `Data\Levels` for level data.
- `Data\BehaviorTrees` for behavior tree data.
- `Data\Scripts` for Lua script data.
- `Data\VPB` for raw protocol-buffer data.

The generated `Tools\build\generator.rb` is the project extension point for
project-specific data rules. Add custom data dependencies there so they feed the
final `nameDataHash.bin` and resource pak generation.

The project `rake data` path generates `_build\win\<build>\build.ninja`, runs
the Ninja `data` alias, and writes runtime data under `_romfiles\win`. A
successful data build ends with `_romfiles\win\nameDataHash.bin`.

## Common Failures

- `USAGI_DIR environment variable not defined`: run `Setup.bat` or
  `EnvironmentSetup.exe` for the current project.
- Ruby gem or Python package failures: confirm Ruby 2.3.3 and Python 2.7.14 are
  first on `PATH`.
- Missing `fbxsdk.h` or `libfbxsdk.dll`: install FBX SDK and set `FBXSDK_DIR`, or
  skip model conversion.
- Missing Oculus projects: configure `OCULUS_SDK_DIR`, or exclude Oculus from
  the project baseline.
- Shader packaging failures: verify effect YAML, shader include paths, and the
  target graphics API argument generated by the rake/Ninja rule.
- Missing runtime data at launch: run `rake platform=win build=debug data` and
  confirm `_romfiles\win` contains `nameDataHash.bin`, shader packages, textures,
  and any resource paks referenced by the project.
